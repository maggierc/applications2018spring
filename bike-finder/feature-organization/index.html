Feature Organization Structure

On Every Page:

  • Cart Button
  • Wishlist Button
    • next to the cart. Represents a page that allows comparison of selected bikes
  • Search Bar
    • includes auto suggestions while typing
  • Home Page Icon
  • FAQ link (in the footer)
    • info on return policies, shipping, etc

Home Page:

  • Large Photos of Bike Types
    • grid of bike-type names centered on home page
    • underneath each title will be a short, quick description to help people quickly understand the differences
  • Dropdown: "Don't Know Your Type?"
    • above grid of bike types
    • user can click on it, and look/choose suggested qualities that narrows down the bike types relevant to them ("multifunctional", "for biking on roads", etc)

List Page:

  • Photo Grid of Bikes
    • small thumbnail of bike
    • name of bike underneath
    • includes extra mark if bike is on sale
    • wishlist button when hovering on photo
  • Filtering
    • left side
    • Bike Type (Cruiser, BMX, etc)
    • Gender Specificity
    • Age Specificity
    • Price
    • Color
    • Gears / Speeds
    • Material
    • Bike Weight
    • Frame Size
    • Wheel Size
    • Attachments (i.e. basket, fenders)
    • Waterproofness
  • Sorting
    • upper right, above the grid
    • Price: low-high
    • Price: high-low
    • Top Customer Review

Detail Page:

  • Photo of Bike
    • zoom-in feature
  • Bike Description
    • paragraph containing extended description
      • dimensions of bike
      • brand
      • warranty coverage
      • possible attachments
      • other tech features
  • bike price
    • w/ estimated shipping
  • Button: Add to Cart
  • Button: Add to Wishlist
  • Customer reviews
  • Safety reviews
  • Alternate Color Options
    • buttons that show the different colors (circular icons?)
  • Other Customers Viewed...
    • image wheel
  • Notes on User Stories:
    • Most aesthetic decisions (color, attached features, comfort options) are made during the refinement phase, with some in the deciding phase
    • Bike type is often decided during the starting and refinement phases
    • Personas tend to want to see customer reviews in the deciding phase, with some choosing to filter by review in refinement
    • Functional factors (material, gears, frame size, etc) are mostly in refinement
    • Personas want the website to offer comparison options in the deciding stage
    • Personas have UI expectations during every stage
    • Price is a general filter in refinement (flat price, sale options); personas want more detailed pricing information in the deciding stage (i.e. how to pay, price comparisons, shipping cost)
